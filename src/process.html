Arbetsprocess

Här beskriver jag arbetsprocessen för utveckling och publicering av denna webbplats.

Utveckling

Webbplatsen är utvecklad med hjälp av Node.js och Parcel som automatiseringsverktyg. Följande process har använts:

Steg för steg-process

  1. En ny katalog skapades för projektet med mkdir web-automation && cd web-automation.
  2. Ett nytt Node.js-projekt initierades med npm init -y.
  3. Parcel installerades som utvecklingsberoende med npm install parcel --save-dev.
  4. Projektets filstruktur skapades, inkluderande mappar för src/, css/, js/ och images/.
  5. En .gitignore-fil skapades för att ignorera node_modules/ och package-lock.json.
  6. För att säkerställa att webbplatsen fungerar på så många webbläsare som möjligt, har jag lagt till browserslist-fältet i package.json. Detta fält definierar vilka webbläsare Parcel ska optimera och transpilera koden för.
  7. Parcel startades för att utveckla webbplatsen med npm run start.
  8. När webbplatsen var klar byggdes den optimerade versionen med npm run build.

Filstruktur

Projektet har följande filstruktur:

  • src/: Innehåller alla utvecklingsfiler.
  • images/: För lagring av bilder.
  • css/: För stilmallar (ex. style.css).
  • js/: För JavaScript-filer (ex. script.js).
  • index.html: Huvudfilen för webbplatsens struktur.
  • .gitignore: Ignorerar oönskade filer i versionshanteringen.
  • package.json: Hanterar beroenden och scripts.

Automatisering

Automatisering implementerades för att förenkla utvecklingsprocessen:

  • Minifiering av CSS, HTML och JavaScript.
  • Optimering av bilder till moderna format som WebP.
  • Automatisk kopiering av optimerade filer till publiceringsmappen.

Publicering

Publicering hanterades med Git och GitHub. Jag har använt två separata branches för att hålla processen organiserad. Development-branch användes under utveckling och testning av nya funktioner, medan Main-branch användes för att hantera publicering av den färdiga versionen. En main-branch används för publicering, och varje commit triggar en automatisk byggprocess via Parcel. Netlify används för att automatiskt bygga och publicera webbplatsen vid varje uppdatering i main.

Verktyg

Följande verktyg användes i projektet:

  • Node.js: Hantering av beroenden och scripts.
  • Parcel: Automatiseringsverktyg för utveckling och produktion.
  • Git och GitHub: Versionshantering och publicering.
  • HTML, CSS och JavaScript: För att skapa strukturen och designen av webbplatsen.
